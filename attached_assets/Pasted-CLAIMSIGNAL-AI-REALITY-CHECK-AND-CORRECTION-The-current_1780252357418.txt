CLAIMSIGNAL AI REALITY CHECK AND CORRECTION

The current app is still displaying “AI” language, but the actual behavior appears to be mostly manual entry, placeholder logic, MVP rule-based checks, and disconnected screens.

This is not acceptable as “AI implemented.”

Do not falsely label anything as AI unless it actually performs model-based extraction, classification, summarization, recommendation, or pattern matching from uploaded claim content.

Do not rebuild the app from scratch.
Do not erase existing data.
Do not remove working features.
Do not weaken login, Master access, RBAC, masking, audit logs, security, claims, evidence upload, adjuster intelligence, carrier intelligence, Playbook, or current routing.

OBJECTIVE

Separate what is truly AI from what is currently manual, placeholder, or rule-based.

Then either:

1. Connect actual AI extraction and analysis where required.

or

2. Clearly relabel non-AI features as MVP rule-based, manual, pending evidence, or pending AI integration.

No fake AI language.

AI TIMELINE REQUIREMENT

Timeline is not correct if it only uses upload date.

AI or extraction logic must identify actual event dates from the uploaded content.

For example:

- denial letter date
- estimate date
- approval letter date
- supplement submission date
- communication date
- reinspection date
- payment date

Audit log date = upload/action date.
Timeline event date = extracted claim event date.

If confidence is low, create:

“Timeline candidate needs review.”

Do not silently insert bad timeline dates.

CLAIM UPLOAD REQUIREMENT

When a file is uploaded, the app should not just store the file.

It should attempt to:

1. Parse text from the file.
2. Extract claim fields.
3. Create or update claim draft.
4. Suggest claim match if claim already exists.
5. Create extraction review card.
6. Create timeline candidates.
7. Create Signal Engine signals.
8. Create missing documentation checklist.
9. Create Playbook recommendation if enough data exists.
10. Log the upload and extraction attempt.

If parsing or AI is unavailable, say:

“File uploaded. AI extraction pending integration.”

or

“File uploaded. Manual review required.”
