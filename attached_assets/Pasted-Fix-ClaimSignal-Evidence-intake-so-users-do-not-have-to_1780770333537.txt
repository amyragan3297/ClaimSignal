Fix ClaimSignal Evidence intake so users do not have to manually extract or manually apply claim data.

Do not redesign the UI.
Do not add demo data.
Do not fake AI results.

Problem:
The Evidence Files section requires too much manual work. The user has to upload a file, manually open AI Extraction Review, manually apply fields, and manually connect the claim. This is not acceptable for ClaimSignal. Evidence intake should automatically extract, match, apply, and link claim data.

Required behavior:
When a user uploads evidence, the system must automatically:

1. Save the file.
2. Detect document type.
3. Run AI extraction.
4. Normalize extracted claim fields.
5. Find or create the canonical claim.
6. Attach the evidence to that claim.
7. Apply extracted fields safely.
8. Link carrier, adjusters, vendors, transcripts, and source references.
9. Show the user a review only when there is a conflict or low confidence.

Evidence upload should not sit there waiting for the user to manually extract the claim.

Required workflow:
Upload evidence
→ AI extraction starts automatically
→ claim matching starts automatically
→ evidence attaches to matched claim automatically
→ safe fields apply automatically
→ relationships link automatically
→ user only reviews conflicts

Confidence rules:
If confidence is high and no conflicts exist:
- auto-apply extraction
- show status: “Auto-applied to claim”

If confidence is medium:
- auto-link evidence to likely claim
- show status: “Needs review”
- allow user to confirm fields

If confidence is low:
- keep evidence in Needs Matching
- show possible claim matches
- allow user to choose existing claim or create new claim

Do not require manual extraction when the system already has claim number, carrier, homeowner, and address.

Required backend changes:
Create or fix pipeline:

processEvidenceUpload(file)

It must call:

extractEvidenceData(file)
findOrCreateCanonicalClaim(extractedData)
applyExtractionToClaim(claimId, extractionData)
linkAdjustersToClaim(claimId, extractionData)
linkCarrierToClaim(claimId, extractionData)
linkVendorsToClaim(claimId, extractionData)
createTimelineEventsFromExtraction(claimId, extractionData)
saveEvidenceSourceLinks(claimId, documentId, extractionId)

Required statuses:
Evidence records should have:
- uploaded
- extracting
- extracted
- auto_matched
- auto_applied
- needs_review
- conflict_review
- failed

UI requirements:
Evidence card should show:
- extraction status
- matched claim
- confidence
- reason matched
- View Extraction
- Review Conflicts only if needed
- View Claim

Replace “Apply to Claim” as the main required action.
It can remain as a fallback button, but it should not be the normal workflow.
