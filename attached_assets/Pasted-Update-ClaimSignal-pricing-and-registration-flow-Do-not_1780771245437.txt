Registration behavior:

Founding Partner:
- User cannot self-activate this plan.
- Clicking Request Founding Partner Access opens a request form, not Stripe checkout.
- Form fields:
  full name
  email
  company
  phone
  reason for early partner access
  expected monthly claim volume
- On submit:
  save request as founding_partner_request
  send/admin surface notification
  show message:
  “Request received. If approved, you will receive access instructions.”
- Do not activate account automatically.

Individual:
- Clicking Get Started opens registration with Individual preselected.
- After registration, create Stripe Checkout session for $129/mo.
- After successful payment, activate account.

Team:
- Clicking Get Started opens registration with Team preselected.
- Seat count must default to 5.
- Minimum seat count is 5.
- Additional seats above 5 billed at $25/user/month.
- After registration, create Stripe Checkout session for $299/mo plus additional seats.
- After successful payment, activate account.

Enterprise:
- Clicking Contact Sales opens sales lead form.
- Do not create Stripe Checkout.
- Do not activate account automatically.

Stripe requirements:
Create or update Stripe products/prices:
- STRIPE_PRICE_INDIVIDUAL_129
- STRIPE_PRICE_TEAM_299
- STRIPE_PRICE_ADDITIONAL_SEAT_25

Do not create a Stripe checkout for Founding Partner unless admin later approves the request.

User status rules:
- pending_billing: registered but Stripe not completed
- active: payment successful
- trialing: active Stripe trial, only if manually approved Founding Partner later
- pending_founding_partner_approval: Founding Partner request submitted
- enterprise_lead: Enterprise sales request submitted
- past_due: payment failed
- canceled: subscription canceled

Access control:

- user status is active
